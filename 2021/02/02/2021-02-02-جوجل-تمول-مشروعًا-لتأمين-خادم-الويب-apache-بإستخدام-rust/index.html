جوجل تمول مشروعًا لتأمين خادم الويب Apache بإستخدام Rust

بتاريخ : 2021/02/02

جوجل تمول مشروعًا في Internet Security Research Group لنقل مكون أساسي من مشروع خادم الويب Apache HTTP من لغة البرمجة C المعرضة للأخطاء إلى بديل أكثر أمانًا Rust.

المكون المعني هو mod_ssl وهو المكون المسؤول عن دعم عمليات التشفير اللازمة لإنشاء اتصالات HTTPS على خادم ويب Apache. حيث قالت ISRG إنها تخطط لتطوير مكون جديد يسمى mod_tls يقوم بنفس الشيء ولكن باستخدام لغة برمجة Rust بدلاً من C.

لقيادة هذا العمل ، تعاقدت إدارة ISRG مع Stefan Eissing ، مؤسس شركة الاستشارات البرمجية Greenbytes ، وأحد ملتزمي أكواد Apache HTTP Server ، لقيادة مشروع mod_tls.

تأمل ISRG أنه بمجرد الانتهاء من عملهم ، سيعتمد فريق خادم الويب Apache HTTP mod_tls كمكون افتراضي واستبدال مكون mod_ssl القديم وغير الآمن.
